Hola Tamara! Tenes una version nueva para probar (0.6.3)

Gracias por el reporte anterior. Arreglamos varias cosas y, sobre todo, agregamos un modo diagnostico que graba lo que el sistema escucha y hace, para entender los puntos de voz que a veces fallan. Te pedimos correr el test una sola vez con ese modo prendido y mandarnos el archivo.

1) Actualiza a la version nueva

Abri la terminal y corre:

npm install -g @yujinapp/yuemail@latest

Para confirmar que quedo la version correcta:

yuemail --version

Tiene que decir 0.6.3. Si dice otra cosa, avisanos antes de seguir.

2) Que ya arreglamos (podes verificarlo de paso)

  • El cuerpo del correo arranca vacio. Antes aparecia un texto puesto por defecto; ya no.
  • Decir solo "fin" (sin agregar "dictado") ahora tambien detiene el dictado.
  • "fin campos" en plural ahora tambien suelta el campo.

3) Prende el modo diagnostico (esto es lo clave)

  1. Abri el panel de Voz.
  2. Bien abajo vas a ver una seccion nueva con un interruptor: "Grabar lo que se escucha y lo que hace la app". Prendelo.
  3. Corre el Test 8 (Dictar texto) y tambien el flujo de envio de correo, una sola vez, hablando normal como siempre.
  4. Cuando termines, apreta el boton "Exportar diagnostico". Te baja un archivo.
  5. Mandanos ese archivo.

Importante: el modo diagnostico no manda nada solo. Lo unico que sale de tu compu es el archivo que vos nos mandas. Y queda prendido aunque cierres y vuelvas a abrir, asi que cuando termines podes apagarlo.

4) Que vamos a poder ver con ese archivo

Por cada cosa que dijiste, vamos a ver exactamente que escucho el sistema, en que pantalla estabas y que decidio hacer. Con eso cerramos los puntos que todavia fallan a veces (el "fin", el "fin campo" que no siempre agarra, y lo del titulo) con datos reales en la mano, sin adivinar.

Cualquier duda, escribinos. Gracias!
