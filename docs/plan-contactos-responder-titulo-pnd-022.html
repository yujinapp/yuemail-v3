Pedido de Pablo sobre el reporte de Tamara. Verifiqué contra el código real: hoy no existe el comando "responder", no existe ninguna lista de contactos, y el armado del destinatario efectivamente se queda con el pedacito pegado a la arroba. Todo lo de abajo es nuevo o un arreglo.

La idea central: el nombre, no el deletreo

Una persona con dificultad de habla no debería tener que decir "eme a equis i... arroba gmail punto com". Debería decir "enviar a Maximiliano" y listo. Para eso construimos una agenda de contactos y hacemos que la app entienda nombres, no direcciones deletreadas.

Qué voy a construir

  1. Agenda de contactos persistente. Una lista guardada en la máquina (nombre + correo, con apodos opcionales) que sobrevive a cerrar la app.
  2. Auto-registro de remitentes. Cada vez que se lee la bandeja, quien escribió queda guardado solo en la agenda. La lista se llena sola con el uso.
  3. Entender el nombre dicho. Cuando la persona dice "enviar a Maximiliano", el asistente busca en la agenda, elige el contacto más parecido y usa SU correo. Si no hay nadie parecido, avisa en vez de inventar una dirección. Esto además mata el bug del destinatario mal armado.
  4. Comando "responder". Después de leer la bandeja, decir "responder" abre el envío ya dirigido a quien escribió el último correo leído.
  5. Dictar el título del documento. Un comando nuevo para ponerle título al documento por voz (hoy no se puede; era el punto que Tamara no encontraba).
  6. Alta de contactos por texto. Una pantallita donde un asistente puede cargar la agenda escribiendo (nombre + correo), para que después la persona se maneje sola por voz. Con su comando de voz para abrirla.
  7. Formateo robusto del correo dictado para cuando sí se dicta una dirección entera.

Cómo encaja con lo que ya hay (sin romper nada)

La app ya usa la técnica correcta: el asistente elige un comando de una lista cerrada. Le voy a sumar la agenda a esa lista para que sepa resolver nombres, y agrego los comandos nuevos (responder, título, abrir contactos) respetando la simetría que el proyecto ya verifica solo. Cero deuda técnica, con pruebas nuevas para cada pieza.
